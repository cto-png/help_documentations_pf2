# ActiveUsers

{% hint style="info" %}
Важно! Для начала зарегистрируйтесь в сервисе [**Active Users**](http://activeusers.ru/)

А для последующих действий необходимо иметь оплаченный тариф и подключенное сообщество в сервисе Active Users.

Посмотреть список тарифов можно [**здесь**](https://activeusers.ru/cabinet/?act=pay#panel1)**.**

Посмотреть инструкцию по подключению сообщества в личном кабинете Active Users - [**здесь**](https://vk.com/@activeusers_ru-2-kak-podkluchit-soobschestvo)**.**
{% endhint %}

* Включите тумблер «Уведомления о разовых оплатах».&#x20;

## 2. Для начала необходимо создать в сервисе валюту, в которой будет производиться оплата.

В нашем случае это рубль. В сервисе Active Users это называется Ресурс. \
Авторизуйтесь и перейдите на вкладку “Ресурсы” и нажмите на кнопку “+ Ресурс”:

<figure><img src="../../../.gitbook/assets/1 (15).png" alt=""><figcaption></figcaption></figure>

Далее заполняем поля:\
\
1\) Поле “Название” - его будете видеть только вы. Так как мы создаем эквивалент рубля, для удобства так и напишем - “Рубль”. \
\
2\) Поле “Описание” - также отображается только администратору в личном кабинете. Здесь вы можете указать свои заметки. Не обязательное поле. \
\
3\) Поле “Тип” - так как мы создаем валюту, то выбираем из списка “Деньги и ключевые ресурсы”. \
\
4\) Поле “Платформа” - выбираете платформу своего сообщества. В нашем случае это будет ВКонтакте.&#x20;

5\) Нажмите на кнопку “Расширенные настройки”.\
\
6\) Поле “Спец. признак” - выбираем из списка значение “Эквивалент рубля”\
\
7\) Поле “Цена” - указываем значение “1”.\
\
8\) Поле “Проекты” - оставляем пустым, так как у нас еще нет проектов. Позже вернемся к этому пункту. \
\
9\) Сохраняем настройки.&#x20;

<figure><img src="../../../.gitbook/assets/2 (15).png" alt=""><figcaption></figcaption></figure>

## 3. Подключение Prodamus в личном кабинете сервиса Active Users.

Перейдите во вкладку "Интеграции" и нажмите "+ Интеграция":

<figure><img src="../../../.gitbook/assets/3 (12).png" alt=""><figcaption></figcaption></figure>

Далее заполняем поля: \
1\) Поле “Тип” выбираем из списка "Продамус"\
\
2\) Поле “Название” - указываем название интеграции - его будет видеть только администратор в личном кабинете Active Users.\
\
3\) Поле “Имя аккаунта”. В пункте 1.1. вы скопировали адрес платежной страницы, например, [https://edu2.payform.ru/](https://edu2.payform.ru/). Именем аккаунта будет та часть, которая располагается до “payform.ru”. В нашем случае это “edu2”. У вас это имя будет другое. \
\
4\) Вводим скопированный секретный ключ (пункт 1.2.)\
\
5\) Поле “Назначение платежа” - указываем за что платит клиент. Это поле вы будете видеть в уведомлениях об оплате. Для примера напишем, Active Users. \
\
6\) Поле “Редирект после оплаты” - на какую страницу необходимо перенаправить пользователя после успешной оплаты. Не обязательное поле, можете не указывать ссылку. \
\
7\) Поле “Вступительный текст” - этот текст будет видеть пользователь перед оплатой. Можно оставить пустым, или указать, например, “Форма оплаты”. \
\
8\) Поле “Проект” - сообщество, в котором будет активен Prodamus. Пока оставим пустым, в следующих шагах укажем проект. \
\
9\) Поле “Активно” - переключить ползунок на “Да”\
\
10\) Обязательно сохраняем.

<figure><img src="../../../.gitbook/assets/4 (12).png" alt=""><figcaption></figcaption></figure>

## 4. Создадим проект для работы в системе Active Users:

Проект - это фильтр ваших приложений в личном кабинете. Если у вас разные сценарии и программы, то, создав проект, вы можете фильтровать какие страницы будут отображаться в каждом проекте. <br>

Для этого перейдите во вкладку "Проекты" и нажмите кнопку "+ Проект":

<figure><img src="../../../.gitbook/assets/5 (11).png" alt=""><figcaption></figcaption></figure>

Вы можете подключать сколь угодно проектов. Например, если вы принимаете оплату на разные ресурсы. Для начала создадим 1 проект и будем с ним работать.&#x20;

Заполняем поля:\
\
1\) Поле “Название” - этот текст будете видеть только вы. Назовем наш проект “Тест”.\
\
2\) Поле “Описание” - этот будете видеть только вы. Это поле не обязательно заполнять. \
\
3\) Поле “Поддомен” - произвольное название адреса вашего проекта. Выбираете имя поддомена, которое вам нравится. \
\
Если вы пропишете имя поддомена, то далее страницу вашего проекта можно будет открыть вне ВКонтакте, а именно, по адресу [поддомен.activeusers.ru](http://activeusers.ru). Другими словами, это просто способ открыть страницу проекта за пределами Вконтакте. \
\
Не обязательное поле, мы оставим его пустым. \
\
4\) Поле “Главная страница” - заполняете в случае, если у вашего проекта несколько страниц. Дополнительные страницы можно создать позже в разделе [Библиотека](https://activeusers.ru/cabinet/?act=vfs\&node=127807). Также не обязательное поле, пока оставляем пустым. \
\
5\) Откройте вкладку “Связанные элементы”\
\
6\) Поле “Сообщества” - к каким сообществам относится этот проект. Выбирайте нужное вам сообщество.&#x20;

7\) Поле “Ресурсы” - выбираем из списка нужную валюту, в нашем случае это “Рубль”.\
\
8\) Поле “Рейтинги” - какие рейтинги относятся к этому проекту. Если у вас еще нет созданных рейтингов, то можно оставить поле пустым и заполнить позже. \
\
9\) Поле “Рассылки” - какие публичные рассылки относятся к этому проекту. Если у вас еще нет созданных рассылок, то можно оставить поле пустым и заполнить позже. \
\
10\) Поле “Страницы” -  это страницы из вашей библиотеки, которые привязаны к проекту, который вы сейчас создаете. Эти страницы, например, “Статистика”, “Достижения” - будут доступны для клиентов. Их также можно создать позже и оставить поле пустым. \
\
11\) Поле “Платежные системы” - выбираем в списке Продамус. \
\
12\) Сохраняем настройки.&#x20;

## 5. Создание товара для покупки

Перейдите во вкладку “Ресурсы” и нажмите на кнопку “+ Ресурс”:

<figure><img src="../../../.gitbook/assets/1 (15).png" alt=""><figcaption></figcaption></figure>

Далее заполняем поля:\
\
1\) Поле “Название” - будет видеть клиент в приложении. Для примера, укажем “Товар”.\
\
2\) Поле “Описание” - это поле будете видеть только вы. Не обязательное поле. \
\
3\) Поле “Тип” - выбираем из списка “Инвентарь и товары”\
\
4\) Поле “Платформа” -  выбираете платформу своего сообщества. В нашем случае это будет ВКонтакте.\
\
5\) Поле “Проекты” - проект, в котором будет отображаться ваш товар.\
\
6\) Нажмите на кнопку “Расширенные настройки”\
\
7\) Поле “Массовость” - выбираем из списка “Одиночные предметы (награды, артефакты, сертификаты, товары)”. \
\
8\) Откройте вкладку “Каталог”\
\
9\) Поле “Показать в каталоге” - нужно переключить ползунок на “Да”\
\
10\) Раздел “Способы получения” - кликните “Добавить новый способ”. \
\
11\) Раздел “Способы получения” - Поле “Ресурсы” - выбираем созданную валюту - “Рубль”.\
\
12\) Раздел “Способы получения” - Поле “шт” - указываем стоимость товара.&#x20;

{% hint style="danger" %}
Важно! Минимальная сумма для оплаты 50 рублей.
{% endhint %}

11\) Раздел “Способы получения” - Поле “Кнопка” - заполняем текстом “Оплатить”\
\
12\) Раздел “Способы получения” - Поле “Ссылка” - если вам необходимо перенаправление после покупки - укажите здесь адрес страницы. Поле не обязательное. \
\
13\) Раздел “Способы получения” - Поле “Программа” - процесс, который запускается после покупки товара: например, открытие доступов, отправка сообщений, начисление бонусов. Если вам необходима программа, то посмотреть как ее настроить можно здесь. \
\
14\) Сохраняем настройки.&#x20;

## 6. Установка приложения Active Users в сообщество

Перейдите в раздел “Мои сообщества”, далее нажмите на шестеренку настроек нужного вам сообщества.\
\
Так вы попадете в настройки сообщества, где можно будет установить приложение Active Users.

<figure><img src="../../../.gitbook/assets/10 (8).png" alt=""><figcaption></figcaption></figure>

Затем откройте вкладку “Приложение” и нажмите на кнопку “Установить VK Mini Apps” - эта версия адаптирована и для компьютеров, и для мобильных устройств:

<figure><img src="../../../.gitbook/assets/11 (6).png" alt=""><figcaption></figcaption></figure>

После того, как вы нажмете на кнопку “Установить универсальное приложение”, откроется список сообществ, в которых вы являетесь администратором. Выберите из списка нужное вам сообщество:

<figure><img src="../../../.gitbook/assets/12 (2).png" alt=""><figcaption></figcaption></figure>

После установки, приложения в вашем сообществе появится приложение “Активные пользователи”. \
\
Для того, чтобы сервис Active Users считал информацию об установке приложения в сообщество, необходимо открыть это приложение в группе ВК:

<figure><img src="../../../.gitbook/assets/13 (2).png" alt=""><figcaption></figcaption></figure>

Достаточного простого захода в приложение, чтобы сервис считал информацию о подключении. Никаких действий в приложении “Активные пользователи” совершать не нужно. \
\
Затем переходим в личный кабинет Active Users в раздел “Мои сообщества” - - “Настройки”:

<figure><img src="../../../.gitbook/assets/10 (8).png" alt=""><figcaption></figcaption></figure>

Далее переходим в раздел “Приложение”, чтобы убедиться, что интеграция с сообществом прошла успешна. В этом разделе должны появиться настройки вида:

## 7. Заключительные настройки интеграции.

Переходим в раздел “Каталог приложений” и находим в списке приложение Prodamus:

<figure><img src="../../../.gitbook/assets/15 (2).png" alt=""><figcaption></figcaption></figure>

После клика по приложению нажмите “Установить”:

<figure><img src="../../../.gitbook/assets/16 (2).png" alt=""><figcaption></figcaption></figure>

А затем выберите из списка необходимое сообщество и снова нажмите кнопку “Установить”:

<figure><img src="../../../.gitbook/assets/17 (2).png" alt=""><figcaption></figcaption></figure>

Далее перейдите в раздел “Мои сообщества”, откройте “Настройки” и перейдите на вкладку “Активности”:

Переключите статус на “Вкл”

<figure><img src="../../../.gitbook/assets/18 (4).png" alt=""><figcaption></figcaption></figure>

Затем кликните на кнопку “Продамус платежи”:

И найдите в блок-схеме “Внешний расчетный модуль” - сокращенно в схеме он подписан, как “ВРМ Продамус”:

<figure><img src="../../../.gitbook/assets/19 (2).png" alt=""><figcaption></figcaption></figure>

После клика по ВРМ откроются настройки, в которых необходимо заполнить поля:\
\
1\) Поле “Платежная система” - выберите из списка “Продамус”.\
\
2\) Поле “Телефон” - обязательное поле. Вставьте значение:\
`{b.0.request.customer_phone}` вставьте значение именно в этом виде, со скобками\
\
3\) Поле “Сумма” - можно указать фиксированное значение цифрами, можно указать в виде переменной, если у вас настроены другие процессы и вы хотите, чтобы сумма генерировалась автоматически под каждый случай покупки:\
`{b.0.request.sum}` переменная суммы\
\
4\) Поле “Описание” - не обязательное, эти данные поступают как дополнительные в систему уведомлений о совершившихся платежах\
\
5\) Сохраняем настройки.

Перед тестированием убедитесь, что ваша платежная форма работает в ДЕМО-режиме.
{% endhint %}

Для теста сначала перейдите в приложение “Активные пользователи” в вашем сообществе:

<figure><img src="../../../.gitbook/assets/13 (2).png" alt=""><figcaption></figcaption></figure>

Найдите в каталоге созданный вами товар и кликните по нему:

<figure><img src="../../../.gitbook/assets/21 (2).png" alt=""><figcaption></figcaption></figure>

Нажмите “Оплатить”:

<figure><img src="../../../.gitbook/assets/22 (1).png" alt=""><figcaption></figcaption></figure>

Открывается форма оплаты.&#x20;

Если у пользователя указан номер телефона, то он подгрузится в форму.&#x20;

Если номер телефона в профиле не заполнен и клиент не укажет его на данном этапе, то в следующем шаге - в платежной форме Продамуса, это станет обязательным полем.&#x20;
